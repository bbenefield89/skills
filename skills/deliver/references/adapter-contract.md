# Capability adapter contract

Read bindings from `docs/agents/workflow-adapters.md`. Require `implementation_driver`, `test_driven_development`, `engineering_quality`, `change_review`, and `final_validation`. Resolve language-hygiene and project-setup capabilities from the same registry. Godot additionally requires `gdscript_hygiene` and `project_setup`.

The current review adapter expects a fixed point and separate Standards and Spec inputs. Supply the recorded task starting point, authoritative spec, repository standards, and engineering principles. Keep its two axes and final output intact.

Capability names are stable interfaces. Concrete skill names and commands are replaceable adapters; do not hard-code them into repository policy.
